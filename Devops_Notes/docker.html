Docker & Containerization

A refined guide to packaging, distributing, and orchestrating modern applications using Docker and Docker Compose.

01. What is Docker?

Docker simplifies the complex problem of software distribution by isolating applications and their dependencies into standardized units called containers. Unlike virtual machines, containers share the host's kernel, making them lightweight and fast.

Reliability

Eliminates the "it works on my machine" problem by ensuring consistent environments from development to production.

Efficiency

Containers spin up in milliseconds and utilize significantly fewer resources than traditional virtualization.

02. Dockerfile Mastery

A Dockerfile is the blueprint for your container. It contains a sequential set of instructions that Docker uses to build an image.

Dockerfile 
# 1. Base Image
FROM node:18-alpine

# 2. Setup Working Directory
WORKDIR /usr/src/app

# 3. Handle Dependencies
COPY package*.json ./
RUN npm install

# 4. Copy Codebase
COPY . .

# 5. Runtime Config
ENV NODE_ENV=production
EXPOSE 3000

# 6. Entrypoint
CMD ["npm", "start"]
FROM

Defines the base image. Prefer minimal versions like -alpine for production-ready images.

WORKDIR

Sets the internal directory. Any subsequent command will be relative to this path.

RUN vs CMD

RUN happens during image construction. CMD is the default command executed when a container starts.

COPY

Transfers files from your local machine into the image's filesystem.

03. Orchestration Concept

In a microservices architecture, individual containers need to communicate. Docker Compose allows you to define and manage multi-container applications in a single YAML file.

Database Layer

MongoDB or PostgreSQL running in isolated, persistent containers.

Backend Services

Express or Spring Boot handling business logic and API requests.

Static Frontend

React or Vue served through high-performance Nginx proxies.

04. Docker Compose

Docker Compose uses a declarative YAML format to specify your entire environment: services, networks, and volumes.

docker-compose.yml 
version: "3.8"

services:
  db:
    image: mongo:latest
    volumes:
      - mongo-data:/data/db

  backend:
    build: ./backend
    depends_on:
      - db
    environment:
      - DB_URL=mongodb://db:27017

  frontend:
    build: ./frontend
    ports:
      - "80:80"

05. VPS Deployment

Deploying containerized apps is consistent across any cloud provider. Here is the streamlined process for a Linux-based VPS.

1. Server Preparation

apt update && apt install docker.io docker-compose -y

Ensure the Docker daemon is active: systemctl start docker

2. Execution

docker-compose up -d --build
