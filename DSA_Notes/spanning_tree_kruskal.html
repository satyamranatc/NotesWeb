Minimum Spanning Trees

A complete algorithmic journey through undirected graphs, greedy edge selections, Disjoint Set Union optimization, and Kruskal's algorithm.

01. What is a Spanning Tree?

Imagine a connected, undirected graph $G = (V, E)$. A **Spanning Tree** of $G$ is a subset of $E$ that forms a tree containing every single vertex of $G$. In simple terms, it connects all vertices together without introducing any loops or cycles.

For a graph with $V$ vertices, a spanning tree always has exactly $V - 1$ edges. Adding a single edge creates a cycle; removing a single edge disconnects the graph.

Real-World Application

Connecting houses to a power grid, planning public transit roads, laying down internet fiber-optic connections, or configuring standard Ethernet bridging (Spanning Tree Protocol) to avoid broadcast storms.

Minimum Weight Goal

In a weighted graph, different links have different costs. The **Minimum Spanning Tree (MST)** is the spanning tree where the sum of edge weights is minimized, providing the cheapest connection blueprint.

02. Kruskal's Greedy Philosophy

Kruskal's algorithm relies on a **greedy heuristic** to build an MST. It treats the vertices as individual isolated elements and grows the tree by selecting the cheapest links one by one.

1

Sort the Edges

Arrange all graph edges in non-decreasing order of their weights. This ensures we are always evaluating the cheapest remaining connections first.

2

Greedy Selection & Cycle Avoidance

Iterate through sorted edges. For each edge, check if its endpoints are already connected. If they are already in the same component, picking it would form a loop—so discard it! Otherwise, pick it and merge components.

3

Terminating Condition

Stop searching immediately once we have selected exactly $V - 1$ edges. The Minimum Spanning Tree is complete.

03. The Engine: Union-Find (DSU)

How do we efficiently check if two vertices are in the same component and merge them in near $O(1)$ time? Doing DFS/BFS cycle searches would be $O(V)$, slowing the algorithm down to $O(V \cdot E)$. Instead, we use a **Disjoint Set Union (DSU)** data structure.

1. Path Compression

Every time we traverse up a node to locate its root, we flatten the tree structure by updating its parent pointers to point directly to the root. This keeps set trees extremely flat.

// Find operation with Compression
int find(int i) {
    if (parent[i] == i) return i;
    return parent[i] = find(parent[i]); 
}

2. Union by Rank

When uniting two disjoint subsets, we attach the shallower tree (lower rank) under the root of the deeper tree (higher rank). This prevents the tree from degenerating into long paths.

// Union operation by Rank
void unite(int i, int j) {
    int rootI = find(i), rootJ = find(j);
    if (rootI != rootJ) {
        if (rank[rootI] < rank[rootJ]) parent[rootI] = rootJ;
        else if (rank[rootI] > rank[rootJ]) parent[rootJ] = rootI;
        else { parent[rootJ] = rootI; rank[rootI]++; }
    }
}

05. Clean Implementation

Here is the highly optimized implementation of Kruskal's algorithm with Disjoint Set Union utilizing Rank and Path Compression.

kruskal.py 
class DisjointSet:
    def __init__(self, vertices):
        self.parent = {v: v for v in vertices}
        self.rank = {v: 0 for v in vertices}

    def find(self, item):
        # Path compression
        if self.parent[item] != item:
            self.parent[item] = self.find(self.parent[item])
        return self.parent[item]

    def union(self, set1, set2):
        root1 = self.find(set1)
        root2 = self.find(set2)

        if root1 != root2:
            # Union by rank
            if self.rank[root1] > self.rank[root2]:
                self.parent[root2] = root1
            elif self.rank[root1] < self.rank[root2]:
                self.parent[root1] = root2
            else:
                self.parent[root2] = root1
                self.rank[root1] += 1
            return True
        return False

def kruskal(vertices, edges):
    # Sort edges by weight
    sorted_edges = sorted(edges, key=lambda edge: edge[0])
    
    dsu = DisjointSet(vertices)
    mst = []
    mst_weight = 0
    
    for weight, u, v in sorted_edges:
        if dsu.union(u, v):
            mst.append((u, v, weight))
            mst_weight += weight
            if len(mst) == len(vertices) - 1:
                break
                
    return mst, mst_weight
Kruskal.java 
import java.util.*;

class Kruskal {
    static class Edge implements Comparable<Edge> {
        int src, dest, weight;
        public Edge(int src, int dest, int weight) {
            this.src = src;
            this.dest = dest;
            this.weight = weight;
        }
        @Override
        public int compareTo(Edge other) {
            return Integer.compare(this.weight, other.weight);
        }
    }

    static class DisjointSet {
        int[] parent, rank;
        public DisjointSet(int n) {
            parent = new int[n];
            rank = new int[n];
            for (int i = 0; i < n; i++) parent[i] = i;
        }
        public int find(int i) {
            if (parent[i] != i) {
                parent[i] = find(parent[i]); // Path compression
            }
            return parent[i];
        }
        public boolean union(int i, int j) {
            int rootI = find(i);
            int rootJ = find(j);
            if (rootI != rootJ) {
                if (rank[rootI] < rank[rootJ]) parent[rootI] = rootJ;
                else if (rank[rootI] > rank[rootJ]) parent[rootJ] = rootI;
                else { parent[rootJ] = rootI; rank[rootI]++; }
                return true;
            }
            return false;
        }
    }

    public static List<Edge> solve(int vertices, List<Edge> edges) {
        Collections.sort(edges);
        DisjointSet dsu = new DisjointSet(vertices);
        List<Edge> mst = new ArrayList<>();
        for (Edge e : edges) {
            if (dsu.union(e.src, e.dest)) {
                mst.add(e);
                if (mst.size() == vertices - 1) break;
            }
        }
        return mst;
    }
}
kruskal.cpp 
#include <iostream>
#include <vector>
#include <algorithm>

using namespace std;

struct Edge {
    int src, dest, weight;
    bool operator<(const Edge& other) const {
        return weight < other.weight;
    }
};

class DisjointSet {
    vector<int> parent, rank;
public:
    DisjointSet(int n) {
        parent.resize(n);
        rank.resize(n, 0);
        for (int i = 0; i < n; i++) parent[i] = i;
    }
    int find(int i) {
        if (parent[i] != i) {
            parent[i] = find(parent[i]); // Path compression
        }
        return parent[i];
    }
    bool unite(int i, int j) {
        int rootI = find(i);
        int rootJ = find(j);
        if (rootI != rootJ) {
            if (rank[rootI] < rank[rootJ]) parent[rootI] = rootJ;
            else if (rank[rootI] > rank[rootJ]) parent[rootJ] = rootI;
            else { parent[rootJ] = rootI; rank[rootI]++; }
            return true;
        }
        return false;
    }
};

vector<Edge> kruskal(int n, vector<Edge>& edges) {
    sort(edges.begin(), edges.end());
    DisjointSet dsu(n);
    vector<Edge> mst;
    for (const auto& edge : edges) {
        if (dsu.unite(edge.src, edge.dest)) {
            mst.push_back(edge);
            if (mst.size() == n - 1) break;
        }
    }
    return mst;
}

06. Mathematical Complexity

Let $E$ represent the total edges and $V$ represent the vertices in the Graph.

Metric Complexity Dominant Factors & Explanation
Time Complexity O(E log E) Dominated by the edge sorting phase. DSU updates using Rank + Path Compression cost $O(E \cdot \alpha(V))$, which is virtually constant. Since $E \le V^2$, we can also express this as $O(E \log V)$.
Space Complexity O(V + E) Required to store DSU structures (ranks, parent arrays) of size $O(V)$ and keep copy array of sorting edges of size $O(E)$.

07. Kruskal's vs Prim's

Choose the optimal Minimum Spanning Tree algorithm for your specific graphical distribution.

Kruskal's Algorithm

  • Edge-Centric: Builds MST by grabbing cheapest edges.
  • Sparse Graphs: Highly optimized for sparse structures ($E \approx V$).
  • Data Structure: Powered by Disjoint Set Union (DSU).
  • Unconnected Support: Easily creates spanning forests for disconnected vertices.

Prim's Algorithm

  • Vertex-Centric: Grows tree outward from a source node.
  • Dense Graphs: Exceptional performance on dense graphs ($E \approx V^2$) when using Fibonacci Heaps.
  • Data Structure: Powered by Priority Queues (Min-Heap).
  • Connectivity: Must traverse only fully connected structures.
